# Z-Index Hierarchy

Иерархия слоев (z-index) в браузере XOLO для правильного отображения элементов интерфейса поверх веб-страниц.

## Структура слоев

### Базовые слои (0-100)
- **0-1**: Фон приложения и основной контент
- **1**: Область веб-страниц (`.main-area`, `.webview-area`)
- **10**: Стартовая страница (`.start-page`)
- **100**: Адресная строка (`.address-bar`)

### UI элементы браузера (500-1000)
- **500**: Внутренние страницы браузера
  - История (`.history-page`)
  - Загрузки (`.downloads-page`)
  - Настройки (`.settings-page`)
- **1000**: Боковая панель (`.zen-sidebar`)

### Модальные окна и оверлеи (10000+)
- **10000**: Оверлеи модальных окон
  - `.new-tab-modal-overlay`
  - `.tab-search-overlay`
  - `.import-dialog-overlay`
- **10001**: Контейнеры модальных окон
  - `.new-tab-modal-container`
  - `.tab-search-container`
  - `.import-dialog-container`
- **10002**: Выпадающие списки внутри модалок
  - `.new-tab-modal__engine-dropdown`
- **10000**: Приветственная страница (`.zen-welcome-page`)

### Уведомления (11000+)
- **11000**: Toast уведомления (`.toast`)

## Правила использования

1. **Веб-страницы всегда внизу** - z-index 1 или ниже
2. **UI элементы браузера выше страниц** - z-index 100+
3. **Модальные окна выше всего UI** - z-index 10000+
4. **Уведомления выше модалок** - z-index 11000+

## Примечания

- Все элементы с z-index должны иметь `position: relative`, `absolute` или `fixed`
- При добавлении новых компонентов следуйте этой иерархии
- Избегайте использования очень больших значений z-index (>20000)
- Используйте `isolation: isolate` для контейнеров webview, чтобы изолировать их контекст наложения
- Адресная строка имеет полупрозрачный фон с backdrop-filter для видимости поверх страниц
- Сайдбар имеет `pointer-events: auto` для корректной работы поверх страниц
